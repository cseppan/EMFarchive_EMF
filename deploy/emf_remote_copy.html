Setup for Remote-to-Local Data Transfer

 

EMF has some features that need to copy files from server to client for view, for example, to view the QA step result on a client machine. For this process to happen, the current EMF user has to verify his/her credentials (public/private keys) with the server so that the secure copy tool can be used to achieve the purpose. Currently, this setup is needed ONLY if one wants to view results from a QA step in the Analysis Engine and/or save to your local computer.

 

This document describes two common remote file copying setups based on a Linux EMF server. EMF client can be run from either a Linux workstation or a Windows workstation. The third configuration is for the setup of an EMF server running off of a local Windows machine.

 

Windows XP

Linux

Client/Server on Windows

 

Update the EMFPrefs.txt File:

 

The following two new items are needed in this file (the values after = may vary depended on your actual local settings):

 

remote.copy.program= pscp.exe

local.temp.dir=C:/temp

 

If there is any space in your directory, please add double quotes to the part that has spaces for the remote.copy.program item and leave it as is for the local.temp.dir item. Here is a good example:

 

remote.copy.program=C:/”Program Files”/PuTTY/bin/pscp.exe

local.temp.dir=C:/Documents and Settings/emf

 

If the pscp.exe file path is in the system %PATH% variable (one can check this by typing “pscp” at any Windows command line and he/she should see “PuTTY Secure Copy client” in the print out), then the following will work for the purpose:

 

remote.copy.program= pscp

local.temp.dir=C:/Documents and Settings/emf

 

Windows XP

 

PuTTY:

 

There are multiple versions of SSH client tools packages available for Windows platforms. The following steps describe the setup of a common SSH client tool package (PuTTY).

 

First, please go to the PuTTY download page (http://www.chiark.greenend.org.uk/~sgtatham/putty/download.html) to download pscp.exe, pageant.exe, and puttygen.exe and put it in a proper directory (e.g. C:\local\bin). You need to make this path known to your system by adding the directory to your %PATH% environment variable. (To add C:\local\bin directory where PuTTY executable files sit into the %PATH% environment variable, one may follow these steps: on the Windows desktop, use mouse to click on “My Computer” icon so that a file browser opens; right click on the back ground of the file browser and click on the “Properties” menu; on the pop-up “System Properties” window, click the “Advanced” tab; in the “Advanced” tab, click on the “Environment Variables” button; in the pop-up “Environment Variables” window, highlight and edit the variable “PATH” either from the “User variables…” pane or the “System variables” pane and append the value “;C:\local\bin” to the end of the “Variable value” field).

 

After the download, you may set up your remote access to EMF server by doing the following:

 

1. Generate private/public RSA key pair

 

Either double click puttygen.exe file link or run puttygen.exe from a Windows command line. This should give you a pop-up window from where you can select RSA key type and key length (default to 1024) and click the Generate button to generate the key pair files.

 

With the PuTTY Key Generator window open, enter the passphrase in the two spaces provided (“Key passphrase” and “Confirm passphrase”) and click “Save private key” button to save the private key into a file. Please memorize the pass phrase because it is required for step 3. It is optional to save the public key file by clicking the “Save public key” button (no pass phrase is required to save the public key file), but it is not used in this case. Then save public key content into a text file by copy & paste the text area from the Key panel following the title “Public key for pasting into OpenSSH authorized_keys file:” in the PuTTY Key Generator window. Both the public key and private key files can be saved to anywhere in your local computer with or without any extension, but user should use his/her cautions so that they are not shared to any other users.

 

2. Copy public RSA key file to EMF server

 

If it is the first time user ever login onto the remote EMF server, he/she needs to get a Windows command prompt and login to EMF server remotely by running putty.exe file (can be downloaded from the same place as pscp.exe is downloaded). Follow these steps to get the command prompt window:

 

Start -> All Programs -> Accessories -> Command Prompt

 

After getting the command prompt, change directory to the place where putty.exe file is at and type:

 

putty.exe USER_NAME@EMF_SERVER_ADDRESS

 

User may be prompted by the system to enter “y” if the user trusts that the server is the one the user thought it was. Simply type “y” and then type the user’s EMF server account password. After the remote login session is open for the user, he/she needs to manually create the “.ssh” directory directly under “/home/USER_NAME” directory if it doesn’t exist. To check this, type “cd” at the command line, then hit Enter key. Type “ls –al” at the command line, then hit Enter key. If the following line appears among the others, it indicates that the .ssh directory exists:

 

drwx------  2 USER_NAME USER_NAME 4096 2007-07-27 10:30 .ssh

 

If .ssh directory doesn’t exist, type the following at the command line to create it:

cd (Enter)

mkdir .ssh (Enter)

 

After doing all these, user may log out of the EMF server by typing “exit” from the remote login session window.

 

Run pscp.exe from a Windows command line by typing the following:

C:\local\bin\pscp.exe PUBLIC_KEY_FILE USER_NAME@EMF_SERVER:/home/USER_NAME/.ssh/authorized_keys2

 

You will be prompted for a password. Enter your EMF server account password to transfer this file to the EMF server.

 

3. Add private RSA key file to SSH key agent

 

Run pagent.exe from a Windows command line by typing the following:

C:\local\bin\pagent.exe PRIVATE_KEY_FILE

 

A pop-up window will ask you for the pass phrase you have put on this file. After you input the correct pass phrase, the pagent logo will show up on your system task bar.

 

From this point on, you don't need to type your EMF server user account password to transfer files or to open an SSH session (you need putty.exe to do this).

 

Linux

 

Since most Linux distributions have SSH client tools installed, there is no need to install them from scratch. In case you don't have them installed, you may download an OpenSSH package from http://www.openssh.com/ for your workstation and follow the instruction to install thereafter.

 

Follow the following steps to set up your key-based SSH authentication to your EMF server once you installed the SSH client tools properly ($ is the bash shell command line prompt in the following steps).

 

1. Generate private/public RSA key pair:

 

$ ssh-keygen

Generating public/private rsa key pair.

Enter file in which to save the key (/home/emf/.ssh/id_rsa):

/home/emf/.ssh/id_rsa already exists.

Overwrite (y/n)? y

Enter passphrase (empty for no passphrase):

Enter same passphrase again:

Your identification has been saved in /home/emf/.ssh/id_rsa.

Your public key has been saved in /home/emf/.ssh/id_rsa.pub.

The key fingerprint is:

62:f8:2a:ff:bf:d6:0f:87:bc:08:8e:c5:26:15:33:c8 emf@cep00156

$

 

2. Copy public RSA key file to EMF server

 

$ scp /home/USER_NAME/.ssh/id_rsa.pub USER_NAME@remote_linux_server:/home/USER_NAME/.ssh/authorized_keys2

authorized_keys2 100% 396 0.4KB/s 00:00

$

 

3. Add private RSA key file to SSH key agent

 

$ ssh-add

Enter passphrase for /home/emf/.ssh/id_rsa:

Identity added: /home/emf/.ssh/id_rsa (/home/emf/.ssh/id_rsa)

$

 

 

For EMF Linux server admin:

 

Please open /etc/ssh/sshd.config file and set the following values:

 

# What ports, IPs and protocols we listen for

Port 22

Protocol 2

RSAAuthentication yes

PubkeyAuthentication yes

 

How to Install a SSH Server on a Windows 2000 or XP

(1) Login as Administrator

(2a) Create a folder c:\temp\cygwin

(2b) Download cygwin's setup.exe from http://www.cygwin.com/ and save setup.exe in c:\temp\cygwin

Click Start->Run and type c:\temp\cygwin\setup.exe

Click Next and choose Install from Internet on “Choose A Download Source” page

Specify your cygwin Root Directory, and select “All Users” and “Unix/Binary” on “Select Root Install Directory” page

When it asks for "Local Package Directory", type c:\temp\cygwin

Select Direct Connection on “Select Your Internet Connection” page

Choose a download site on the next page.

When a selection screen comes up (you can resize the windows to see better),

Click the little View button for "Full" view ,

·        find the package "openssh", click on the word "skip" so that an appears in Column B

·        find the package "tcp_wrappers", click on the word "skip" so that an appears in Column B

·        find the package "zlib", click on the word "skip" (it should be already selected) so that an appears in Column B

Click Next to start installing cygwin and ssh.

(3) Right click My Computer->Properties->Advanced->Environment Variables

Click the "New" button to add a new entry to System variables:
            variable name is CYGWIN
            variable value is ntsec tty

(4) Right click My Computer->Properties->Advanced->Environment Variables
            Select the Path variable and click the "Edit" button:
            append  ;YOUR_CYGWIN_ROOT_DIR\bin   to the end of the existing variable string.

(5) Open a cygwin window (by double clicking the cygwind icon), a black screen pops open, type
            ssh-host-config     

            When the script asks you about "privilege separation", answer yes
            When the script asks about "create local user sshd", answer yes
            When the script asks you about "install sshd as a service", answer yes
            When the script stops and asks you for "CYGWIN="     your answer is ntsec tty 

(6) While you are still in the (black) cygwin screen, start the sshd service by typing
            net start sshd  or
            cygrunsrv  --start  sshd

(7) Make sure every Windows user has done the following at least once:
            Login in as the Windows user, pop a console command screen by clicking Start->Run->cmd

(8) Open a cygwin window, synchronize Windows user information with cygwin by typing
            mkpasswd   -cl   >   /etc/passwd
            mkgroup   --local   >   /etc/group

(9) Edit the sshd configuration file at YOUR_CYGWIN_ROOT_DIR\etc\sshd_config and save it. Make sure the following items are uncommented:

    

     Port 22

     Protocol 2

 

     HostKey /etc/ssh_host_rsa_key

     HostKey /etc/ssh_host_dsa_key

 

     StrictModes no

 

     RSAAuthentication yes

     PubkeyAuthentication yes

     AuthorizedKeysFile .ssh/authorized_keys2

 

(10) Follow link Windows XP section to set up your public key authentications.
