    4. Linux Manual Client Installation

    The EMF client can be installed on either the same computer as the EMF server or a different computer. In addition, the OS of the client and the server do not need to be the same. The following easy steps should get a EMF client up and running on a Linux machine:

    1. Download and install the latest Java Runtime Environment 6 (e.g. JRE 6u1). See Java Details.

    2. Download and untar EMF Client (EMFClient.tar.gz)

    3. Modify the preference file EMFPrefs.txt. NOTE: the input and output file directories should be directories on the remote EMF server, not on your local client's machine. Examples of fields to modify and potential values:

      4. local.input.drive=/home/emf/data/
remote.input.drive=/home/emf/data/
local.output.drive=/home/emf/data/
remote.output.drive=/home/emf/data/
default.input.folder=input
default.output.folder=output
server.address=http://emf.ie.unc.edu:8080/emf/services

    4. Copy the preference file to your home directory. If you decide to place it somewhere else and/or rename it, you will need to make sure the EMF client script reflects these changes.

    5. Modify the EMFClient.sh or EMFClient.csh script (depending on your shell). For the bash shell, you would modify the top of the EMFClient.sh to reflect your system:

      6. EMF_HOME=~/EMFClient
JAVA_HOME=/usr/local/java/jre1.6.0_01
PREF_FILE=~/EMFPrefs.txt
TOMCAT_SERVER=http://emf.ie.unc.edu:8080

    6. Start the EMF client (remember to use the correct script for your SHELL):

      7. ./EMFClient.sh &

    7. If you want to setup an EMF client icon, right click on the desktop, then click Create Launcher. Set the name to "EMF Client" and browse to the shell script for the Command. There is an icon in the EMFClient directory.

    First Time Linux Server Installation

    Overview

    1. Install the latest Java Runtime Environment 6 (e.g JRE 6u1)
    2. Install PostgreSQL 8.2
    3. Install Tomcat 5.5
    4. Install the EMF database setup script
    5. Setup the server's directory permissions

    Java Details

    Note where java is installed. For Tomcat and the client, you will need to define the top of the java directory tree. For example, in tcsh or csh you would define the top as (depending on your installation location): 

    setenv JAVA_HOME /usr/local/java/jre1.6.0_01
    and in bash: 
    export JAVA_HOME=/usr/local/java/jre1.6.0_01

    If the client and the server are on the same machine, you only need to install Java once. If the client is on a different machine, then Java needs to be on both the client and server machines.

    PostgreSQL Details

    We recommend installing PostgreSQL in /usr/local and placing the data directory in a location where it may grow. Depending on the number of inventories, the PostgreSQL database may expand to a significant size.

    The following commands will likely needed to be done as root or using sudo (depending on your installation location):

    1. Download and untar the PostgreSQL source into a tmp or build directory

    2. Follow the standard PostgreSQL install instructions, install under /usr/local:
      3. ./configure --prefix=/usr/local
gmake
gmake install

    3. Create and initialize the data directory
      4. mkdir /usr/local/pgsql_data
adduser postgres
chown postgres /usr/local/pgsql_data
/usr/local/bin/initdb -D /usr/local/pgsql_data

    4. Modify and add startup script
      5. cp contrib/start-scripts/linux /etc/init.d/postgresql
      Edit new script-- set the prefix (e.g. to /usr/local) and PGDATA (e.g. to /usr/local/pgsql_data) 
      chkconfig --add postgresql
/etc/init.d/postgresql start

    This should create a working PostgreSQL database server that will start automatically when you boot your machine. At this point, the database is empty and only has 1 user, postgres. In the EMF database setup, we will populate the database and create the base user, emf.

    We recommend that you install pgAdmin to help manage your PostgreSQL database. Many Linux package managers include this utility or it can be installed from source.

    Note, if you already have a PostgreSQL 8.2 server for some other non-EMF applications, you do not need to reinstall the server. The following Tomcat and EMF database setup should work with your previous server.

    Tomcat details

    1. Download Tomcat 5.5, specifically the Core tar file.

    2. Untar the Tomcat 5.5 download into your desired directory. We recommend /usr/local.

    3. Within the Tomcat directory, edit the file conf/tomcat-users.xml, add the line (note the password can be changed if you want):
      4. <user username="admin" password="admin" roles="admin,manager"/>

    4. Download the postgres JDBC 3 jar (for PostgreSQL 8.2) and copy the jar file into Tomcat's common/lib

    5. Decide who will run the tomcat apache server:

      6. root
      The advantage of running as root is that you need to make the fewest modifications to the scripts and directory permissions. The disadvantage is that files created by the apache server (e.g. exports from EMF) will have root as the owner. To setup the server to run as root:
      1. Download our tomcat startup script for root

      2. Modify the script so that CATALINA_HOME and JAVA_HOME (see Java details) match your directory structure

      3. Add the startup script to automatic boot sequence and start the server
        4. cp tomcat_root /etc/init.d/tomcat
chkconfig --add tomcat
/etc/init.d/tomcat start

      user
      The advantage of running as a specific user, is that all files created (e.g. exports EMF) will be owned by that user. The disadvantage is that you will need to make sure the user has permissions to write to the tomcat directory and to the input and output directories defined in the EMFPrefs.txt file. In this example, the linux user is emf:
      1. Create user and give them write permissions for the tomcat apache directory:
        2. adduser emf
chown -R root:emf apache-tomcat
chmod -R g+rw apache-tomcat

      2. Download our tomcat startup script for user

      3. Modify the script so that TOMCAT_USER, CATALINA_HOME, and JAVA_HOME (see Java details) match your system

      4. Add the startup script to automatic boot sequence and start the server
        5. cp tomcat_user /etc/init.d/tomcat
chkconfig --add tomcat
/etc/init.d/tomcat start
    6. On the server, you can go to http://localhost:8080/ to access the Tomcat manager.

    7. The Linux administrator will likely need to open port 8080 in the system firewall for the server to accept connections from remote EMF clients.

    EMF database setup details

    To setup the EMF database, you could either create a database that is empty or one which is fully populated with the EMF training data:

    empty
    1. Download and untar EMFServer tar file

    2. Run the setup script. Note, this will create a new PostgreSQL user emf and a new database EMF. The script will ask you for a password for the emf user, this password must be "emf", and it is for the database server, not the Linux user emf (see Tomcat details):
      3. cd EMFServer/db
./SetupEmfSchemasFromScatch.setup

    3. Copy the war file to the apache server and restart the server (assuming that apache is located at /usr/local/apache-tomcat:
      4. cp EMFServer/emf.war /usr/local/apache-tomcat/webapps
/etc/init.d/tomcat restart

    training
    1. Download and untar EMFServer_training tar file

    2. Run the setup script. Note, this will create a new PostgreSQL user emf and a new database EMF. The script will ask you for a password for the emf user, this password must be "emf", and it is for the database server, not the Linux user emf (see Tomcat details):
      3. cd EMFServer_training/db
./SetupEmfTraining.sh

    3. Copy the war file to the apache server and restart the server (assuming that apache is located at /usr/local/apache-tomcat:
      4. cp EMFServer_training/emf.war /usr/local/apache-tomcat/webapps
/etc/init.d/tomcat restart

    Server directory permissions

    For the server to properly import and export datasets, both the tomcat user (in our case "emf") and the postgres user need to have permissions to write to the export directory. In addition, the postgres user needs to have read and execute permissions for all directories in the directory hierarchy. For example, if all exports were written to /data/emf/emf_output, then postgres should have read,execute permissions for /data and /data/emf and both users should have read,write,execute for /data/emf/emf_output. For import, only the tomcat user needs to have read, execute permissions for the import directory.

    Windows Client Installation using Installer

    1. The EMF client runs best under Java 6. Before running the installer or the client, please download and install the latestJava Runtime Environment 6 (e.g. JRE 6u1) from http://java.sun.com/javase/downloads/index.jsp

    2. Download the EMF Installer Executable (emfinst.exe)

    3. Double click on emfinst.exe to run the installer.
      • First, the installer will expand the zipped compressed file onto your hard-disk. The default location is C:\emfinst. This is a very small installation of the installer software only, and does not require much disk space.
      • Once the expansion is complete, the installer program will start.

    4. In the EMF Client Installer window, fill in the fields. You may click Browse to help you choose any local directories. (NOTE: the input and output file directories should be directories on the remote EMF server, not on your local client's machine - the installer will warn you if they don't exist on your client machine, you can ignore that warning and keep installing) Examples of filled-in fields are the following:
      • EMF Download URL:http://www.ie.unc.edu/cempd/projects/emf/install
      • Java Home Directory: C:\Program Files\Java\jre1.6.0_01
      • Input File Directory:T:\emfdata\input
      • Output File Directory:T:\emfdata\output
      • Client Home Directory: C:\Program Files\EMFClient
      • Server Address:http://emf.ie.unc.edu:8080/emf/services

    5. Click Install.

    6. If you already have the client installed, you will be prompted about whether you wish to Update or Reinstall. Update will download only changed files, and Reinstall will download all files. In most cases, you can Update, but if things do not seem to be working properly you can try to Reinstall.

    7. The installer will provide feedback on the files it is downloading. The latest list of installation files is available from files.txt

    8. When the installation is finished, click Done.

    Note: A file containing the preferences (i.e. stored settings) for the installer called EMFUserInstallPrefs.ini is stored in your user home directory (usually C:\Documents and Settings\yourusername), and a file containing the EMF preferences called EMFPrefs.txt in the same directory.

    First Time Windows Server Installation

    Overview

    1. Install the latest Java Runtime Environment 6 (e.g. JRE 6u1). Use the"Typical Setup".
    2. Install PostgreSQL 8.2
    3. Install Tomcat 5.5
    4. Install the EMF database setup script

    Java Details

    Note where java is installed. For Tomcat and the client, you will need to define the top of the java directory tree. Using Control Panel, create a new Environment Variable called JAVA_HOME and set its value to be the location where you installed Java (e.g., C:\Program Files\Java\jre1.6.0_01 )

    If the client and the server are on the same machine, you only need to install Java once. If the client is on a different machine, then Java needs to be on both the client and server machines.

    PostgreSQL Details

    Download the Win32 multi-language version binary installer (zip file). The installation instructions are here. Be aware that several screen shots of windows in this document may not match the actual windows in the Installer.

    Notes:

    1. Make sure to read the "Readme.txt" file completely before installing PostgreSQL.

    2. This is not in the readme file -- the file which opens the application is called "postgresql-8.2.msi", not "postgresql-8.2-int.msi".

    3. PostgreSQL uses two separate logins. You may need to start a service on your computer, using Control Panel, called "Secondary Logon".

    This should create a working PostgreSQL database server that will start automatically when you boot your machine. At this point, the database is empty and only has 1 user, postgres. In the EMF database setup, we will populate the database and create the base user, emf.

    Note: if you already have a PostgreSQL 8.2 server for some other non-EMF applications, you do not need to reinstall the server. The following Tomcat and EMF database setup should work with your previous server.

    Tomcat details

    1. Download Tomcat 5.5, specifically the Core Windows Service Installer.

    2. Run the Windows Installer from your computer.

    3. Be sure to put the installation in a directory name without spaces. For all other options, choose the defaults except choose your favorite admin password.

    4. Download the postgres JDBC 3 jar (for PostgreSQL 8.2) and copy the jar file into Tomcat's common/lib

    Notes:

    1. You should set the Tomcat service to "automatic" mode by using the Control Panel.

    2. You can go to http://localhost:8080/ to access the Tomcat manager.

    3. If you have issues with Tomcat, you should check the logs available under the "logs" folder under Tomcat.

    EMF database setup details

    The last stage is to setup the EMF database. You have two options, you could either create a database that is empty or one which is fully populated with the EMF training data:

    training (recommended setup option)
    1. Download and unzip the EMFServer_training file. In the db subfolder, you will find the SQL setup scripts and restore data.

    2. Open the setup script under db called SetUpEMFTraining.sh. You will run the steps manually (do not run the script directly) using pgadmin as in the next step.

    3. Start pgadmin (it is part of the postgreSQL distribution) and do the following:
      • Create a new user by double clicking "postgreSQL DB Server 8.2", then navigating to login roles and right-clicking "new Login Role". The name of the role must be "emf", with the password of "emf". Click all checkboxes for the role.
      • Create a new database by going to Databases and right-clicking "New Database". The name of the database must be "EMF" (all caps) with an owner of "emf" and a UTF-8 encoding scheme.
      • Using the setup script as a guide, restore the new database by highlighting "EMF", then right-click restore. Navigate to the "emf_training_db_05102007.backup" file in db.

    4. From the Tomcat manager (usually http://localhost:8080), install the EMF WAR file (emf.war) from either the EMFServer folder or a directly download it, emf.war.

    empty
    1. Download and unzip the EMFServer file. In the db subfolder, you will find the SQL setup scripts and restore data.

    2. Open the setup script under db called SetUpEMFSchemasfrom Scratch.setup. You will run the steps manually (do not run the script directly) using pgadmin as in the next step.

    3. Start pgadmin (it is part of the postgreSQL distribution) and do the following:
      • Create a new user by double clicking "postgreSQL DB Server 8.2, then navigating to login roles and right-clicking "new Login Role". The name of the role must be "emf", with the password of "emf". Click all checkboxes for the role.
      • Create a new database by going to Databases and right-clicking "New Database". The name of the database must be "EMF" (all caps) with an owner of "emf" and a UTF-8 encoding scheme.
      • Using the setup script as a guide, copy and run the 7 commands starting with "psql", one after another in the order listed, into the window opened from the database name (i.e., EMF).
      • Using the setup script as a guide, restore the new database by highlighting "EMF", then right-click restore. Navigate to the "emf_reference.backup" file in db.

    4. From the Tomcat manager (usually http://localhost:8080), install the EMF WAR file (emf.war) from either the EMFServer folder or a directly download it, emf.war.
